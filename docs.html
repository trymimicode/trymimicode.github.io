Installation

mimicode works on macOS, Linux, and Windows. Choose your platform:

macOS / Linux

curl -fsSL https://trymimicode.github.io/install.sh | bash

The installer will:

  • Check for Python 3 and pip
  • Install ripgrep (if needed)
  • Clone the repository to ~/.local/mimicode
  • Create a Python virtual environment
  • Install dependencies
  • Add mimicode to your PATH

Windows

irm https://trymimicode.github.io/install.ps1 | iex

The PowerShell installer will:

  • Check Python 3 installation
  • Auto-install ripgrep (via scoop, choco, or direct download)
  • Clone to %LOCALAPPDATA%\mimicode
  • Setup virtual environment
  • Install Python dependencies
  • Create command wrapper and add to PATH

Manual Installation

git clone https://github.com/alvinliju/mimicode
cd mimicode
python3 -m venv venv
source venv/bin/activate  # or: venv\Scripts\activate on Windows
pip install -r requirements.txt
python3 check_deps.py
Requires: Python 3.9+, pip, and ripgrep (rg)

Quick Start

1. Set Your API Key

export ANTHROPIC_API_KEY="sk-ant-..."

On Windows, use:

$env:ANTHROPIC_API_KEY = "sk-ant-..."

2. Run mimicode

mimicode

This starts an interactive REPL session where you can give it tasks.

3. Give It a Task

>> read src/main.py and explain what it does

mimicode will use its tools to complete the task and report back.

Tip: Your conversations are saved to sessions/ and can be resumed later.

CLI Commands

Interactive Mode (REPL)

mimicode

Start an interactive session. Type your task or question at the prompt.

Single Command

mimicode "read src/main.py and explain"

Execute a single task and exit. Perfect for scripts and automation.

Named Session

mimicode -s myproject "add error handling to auth.py"

Create or resume a named session. Sessions persist across runs.

Resume Session

mimicode -s myproject

Continue a previous session in REPL mode.

TUI Mode

mimicode --tui

Full-featured terminal UI with rich formatting and navigation.

List Sessions

mimicode --list-sessions

Show all saved sessions.

Get Help

mimicode --help

Display all available options.

Usage Modes

REPL Mode

Interactive prompt where you can refine tasks step-by-step:

> read src/app.py
> now add a function to handle errors
> create a test file for that
> done

One-Shot Mode

Execute a single task with complete output:

mimicode "fix the TypeError in line 42 of utils.py"

TUI Mode

Full terminal UI with syntax highlighting, session history, and navigation.

Piped Input

Chain mimicode with other commands:

echo "read README.md" | mimicode

Session Management

Sessions automatically save your conversation, allowing you to resume work later.

Session Files

Sessions are stored in the sessions/ directory:

  • sessions/<id>.jsonl — Event stream (metadata)
  • sessions/<id>.messages.json — Full conversation

Create a Named Session

mimicode -s auth-refactor

Resume a Session

mimicode -s auth-refactor

List All Sessions

mimicode --list-sessions

Delete a Session

rm sessions/auth-refactor.*

Philosophy: Deliberate Over Broad

mimicode is built on the principle that less context is better. Instead of loading your entire codebase and asking the model to figure out what matters, mimicode:

  • Requires ripgrep — Fast, focused searches that respect .gitignore
  • Reads before editing — Understands context before making changes
  • Scopes narrowly — Operates on specific files, not entire directories
  • Bans broad operations — No find, no grep -r, no ls -R
  • Caps tool output — 100KB limit forces focused, efficient searches

This approach reduces token waste, speeds up execution, and keeps costs low.

Core Tools

mimicode provides four focused tools:

1. Read

Read file contents with smart truncation. Output capped at 100KB.

read src/app.py
read src/app.py --lines 1-50

2. Write

Create or overwrite files. Requires reading first to verify intent.

write path/to/file.py "file contents"

3. Edit

Make targeted edits to existing files with precise line ranges.

edit src/app.py 42:50 "new code here"

4. Bash

Execute shell commands with safety guardrails. Blocks dangerous operations like rm -rf /.

bash "npm test"
bash "python -m pytest"
Safety: Certain commands are blocked: rm -rf, curl | sh, grep -r, find, cat /etc/passwd, etc.

API & Providers

Anthropic API

mimicode uses the Anthropic API with Claude models. Set your API key:

export ANTHROPIC_API_KEY="sk-ant-..."

Prompt Caching

Automatic prompt caching reduces API costs by reusing cached context across requests. No configuration needed.

Supported Models

  • claude-opus-4-1 — Most capable, higher cost
  • claude-sonnet-4-1 — Balanced performance and cost
  • claude-haiku-3-5 — Fast and cheap

Set the model by editing providers.py or via environment variable:

export ANTHROPIC_MODEL="claude-sonnet-4-1"

Frequently Asked Questions

Q: What's the difference from other coding agents?

A: mimicode is built for efficiency. It uses 75% less context, costs 75% less to run, and completes tasks faster by being deliberate about what it reads and how it operates.

Q: Can I use this for production code?

A: Yes. mimicode respects your code review process — it suggests changes you review before committing. Always test thoroughly.

Q: Is my code sent to Anthropic?

A: Yes, code is sent to Anthropic's API for processing. Use only with code you're comfortable sharing. Don't include secrets or sensitive data.

Q: How do I update mimicode?

A: On macOS/Linux: cd ~/.local/mimicode && git pull. On Windows: Manual update in %LOCALAPPDATA%\mimicode.

Q: Can I use a different AI provider?

A: Currently only Anthropic is supported. OpenAI support is planned.

Q: How do I contribute?

A: Submit issues and PRs to the GitHub repository.

Troubleshooting

Installation Failed: "ripgrep not found"

Install ripgrep manually:

# macOS
brew install ripgrep

# Ubuntu/Debian
sudo apt install ripgrep

# Fedora
sudo dnf install ripgrep

# Windows (Chocolatey)
choco install ripgrep

# Windows (Scoop)
scoop install ripgrep

"ANTHROPIC_API_KEY not set"

Set your API key:

export ANTHROPIC_API_KEY="your-key-here"

Or on Windows:

$env:ANTHROPIC_API_KEY = "your-key-here"

"ModuleNotFoundError: No module named 'anthropic'"

Reinstall dependencies:

pip install --upgrade -r requirements.txt

"Command 'mimicode' not found"

The PATH may not be updated. On Linux/macOS:

source ~/.bashrc  # or ~/.zshrc

Or add manually:

export PATH="$HOME/.local/bin:$PATH"

"rg: command not found" (during operation)

ripgrep is required. Install it using the steps above.

Sessions not saving

Ensure the sessions/ directory exists and is writable:

mkdir -p ~/mimicode/sessions
chmod 755 ~/mimicode/sessions
Still stuck? Open an issue on GitHub.
